  BUG #2 - CRITICAL   semantic_similarity receives raw cross-encoder logits
File: backend/engines/optimized_ranking_engine.py  |  confirmed: semantic_similarity max = 3.05 in ranking_detailed.json
Impact: The scoring formula weights semantic_similarity at 0.01, so the numerical effect is small, but the component value is meaningless (logits are not probabilities or similarities). A cross-encoder score of 3.05 is being added to components that are all in [0,1]. The formula is semantically broken.

The Problem
In rank_candidates_fast, the FAISS score (or cross-encoder raw logit) is passed directly:
semantic_sim = faiss_score_map.get(candidate_id, 0.0)
components = self.feature_scorer.score_candidate(
    candidate, parsed,
    semantic_similarity=semantic_sim,  # raw logit, not in [0,1]
    ...
)

The Fix
Normalize the cross-encoder score with a sigmoid before passing it in. Add this line just before the score_candidate call:
import math

# Normalize raw cross-encoder logit to [0, 1] via sigmoid
raw_sem = faiss_score_map.get(candidate_id, 0.0)
semantic_sim = 1.0 / (1.0 + math.exp(-raw_sem))

components = self.feature_scorer.score_candidate(
    candidate, parsed,
    semantic_similarity=semantic_sim,  # now in (0, 1)
    ...
)

  BUG #4 - HIGH   Score ceiling collapse - 8-way tie at 1.0000
File: backend/engines/feature_scorer.py  |  Property: ScoringComponents.final_score
Impact: 8 of your top candidates are indistinguishable at score=1.0000. Judges reviewing your JSON see no differentiation at the very top. The score formula correctly computes higher values (base × 1.3 behavioral easily exceeds 1.0) but clamps them. You're throwing away ordering information for your best candidates.

The Fix - Two-Step Normalization
Step 1: Remove the hard clamp from ScoringComponents.final_score - let it compute the true value:
@property
def final_score(self) -> float:
    base = (
        self.title_relevance        * 0.30 +
        self.skill_trust_score      * 0.25 +
        self.assessment_score       * 0.20 +
        self.experience_level_fit   * 0.08 +
        self.education_score        * 0.05 +
        self.technical_relevance    * 0.05 +
        self.production_experience  * 0.04 +
        self.evaluation_framework_score * 0.02 +
        self.semantic_similarity    * 0.01
    )
    raw = base * self.profile_quality_multiplier * self.behavioral_multiplier
    return max(raw, 0.0)  # Remove the min(raw, 1.0) clamp here

Step 2: After collecting all scored candidates in rank_candidates_fast, normalize by dividing by the maximum raw score:
# After scoring all candidates, before sorting:
if scored_candidates:
    max_raw = max(s['final_score'] for s in scored_candidates)
    if max_raw > 0:
        for s in scored_candidates:
            s['final_score'] = s['final_score'] / max_raw

# Then sort as before
scored_candidates.sort(key=lambda x: (-round(x['final_score'], 4), x['candidate_id']))
This preserves all relative ordering, eliminates the ceiling collapse, and gives your best candidate exactly 1.0000 while spreading everyone else proportionally.


  BUG #5 - HIGH   assessment_score = 0.0 for candidates with no keyword-matched assessments
File: backend/engines/feature_scorer.py  |  Method: _score_skill_assessments()
Impact: Your rank-1 candidate (CAND_0002025, Senior AI Engineer at Apple) has assessment_score=0.0 despite being at the top. The 20% weight on assessment_score is the third-largest component. Any candidate whose platform assessment names don't exactly substring-match JD keywords loses 20% of their base score.

The Fix
Add a fallback: when no assessments match JD keywords, return a discounted average of all available assessments rather than zero:
def _score_skill_assessments(self, candidate_raw, jd_skill_keywords):
    assessments = candidate_raw.get('redrob_signals', {}).get(
        'skill_assessment_scores', {}
    )
    if not assessments:
        return 0.0

    relevant_scores = []
    for skill_name, score in assessments.items():
        skill_lower = skill_name.lower()
        if any(jd_kw in skill_lower or skill_lower in jd_kw
               for jd_kw in jd_skill_keywords):
            relevant_scores.append(score / 100.0)

    if relevant_scores:
        return min(sum(relevant_scores) / len(relevant_scores), 1.0)

    # FALLBACK: no keyword match, use discounted average of all assessments
    all_scores = [v / 100.0 for v in assessments.values()]
    return min(sum(all_scores) / len(all_scores) * 0.6, 1.0)  # 40% discount
